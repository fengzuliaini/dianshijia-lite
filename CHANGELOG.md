# 更新日志 (CHANGELOG)

本项目专注于为老电视及机顶盒（Android 4.1+, API 16+）与智能触屏手机提供完美的兼容性、流畅的遥控与触屏操作、以及极致的流媒体播放体验。

---

## [V1.3.3] - 2026-05-27
### 优化 (Improved)
* **老电视 H.265 (HEVC) 直播源兼容性硬伤优化**：
  * **H.265 线路降权排序**：老旧电视（API <= 20）下，若线路 URL 包含 `h265` 或 `hevc` 关键字，在解析源时自动沉底，将 H.264 兼容源线路置顶，实现换台即秒开，无需经历漫长的报错和自动切线等待。
  * **主 M3U8 H.265 自适应子流剔除**：在代理服务重写 M3U8 时，自动过滤含有 `hev1/hvc1/h265` 等 H.265 编码的子流属性行和下一行 URL，防止老电视播放器因硬解不支持而报错。
### 修复 (Fixed)
* **带防盗链 Cookie 会话校验的源代理黑屏**：
  * **全量响应头透传**：在代理服务 `LocalProxyServer` 转发响应时，自动将真实服务器返回的 Headers（除 Content-Length、Content-Type、Connection、Transfer-Encoding 冲突项外，完整透传如 `Set-Cookie` 字段）写回给播放器，建立完整防盗链会话。

## [V1.3.2] - 2026-05-26
### 修复 (Fixed)
* **跨分组同名频道误合并导致的直播源缺失**：
  * **联合 Key 去重**：修改 `M3uParser.java` 去重 Key 逻辑，从之前单纯以 `currentName` 聚合改为按 `currentGroup + "_" + currentName` 唯一聚合。防止不同分组下的同名频道（如不同地方台的综合频道、新闻频道）被错误合并覆盖，找回所有丢失的地方台。
* **特定直播源代理拉流解析失败**：
  * **请求头透传**：重构本地代理服务 `LocalProxyServer` 转发请求头机制。完整读取客户端发来的 Header 并透传给真实远端服务器（排除 Host 头部以防冲突），以兼容特定源的防盗链（Referer/Cookie等）校验和播放器优化逻辑。

## [V1.3.1] - 2026-05-26
### 修复 (Fixed)
* **老电视全量代理黑屏/解析失败故障**：
  * **媒体格式后缀兼容**：对代理后的链接自动提取并保留原始扩展名（如 `/proxy.m3u8` 或 `/proxy.ts`），解决 ExoPlayer 无法识别代理链接媒体类型导致拒绝播放的问题。
  * **请求头缓冲区处理**：服务端增加请求头 Headers 消耗循环，避免因 Headers 滞留导致网络协议栈提前重置连接（Connection Reset）。
  * **短连接策略注入**：在代理响应中强制注入 `Connection: close` 头，指示客户端禁止复用 Socket，契合代理的单次线程处理逻辑。
  * **状态行和异常容错**：防御了 `response.message()` 为 null 的情况，并在代理异常时优雅回吐 `502 Bad Gateway` 状态码。
* **开机自启续播频道匹配失效**：
  * **原始URL存储**：在 `playChannel` 保存历史记录时改存原始 URL 而非代理 URL，修复了老电视（使用代理）重启后无法匹配到上次播放频道的问题。

## [V1.3.0] - 2026-05-26

### 🚀 全量 HTTP 本地代理——彻底修复老电视播放失败
* **HTTP/HTTPS 全量代理突破**：
  * 经数据分析发现所有 300 个直播源 URL 均为 `http://` 协议，此前四轮的 HTTPS-only 代理条件 (`url.startsWith("https://")`) 永远不满足，代理从未被触发。
  * 将 `proxyUrlIfNeeded()` 中的条件扩展为同时匹配 `http://` 和 `https://`，在老设备 (API ≤ 20) 上实现全协议全量代理。
* **302 重定向后相对路径 TS 分片支持**：
  * 分析发现上游服务器会将请求 302 重定向至带一次性 token 的 IP 地址，且 M3U8 内所有 TS 分片均为相对路径。
  * 重写 `rewriteM3u8Content()` 方法，利用 OkHttp `response.request().url()` 获取重定向后的最终 URL 作为 base，逐行解析 M3U8 内容，对绝对路径和相对路径统一改写为本地代理地址。
* **OkHttp 超时防护**：
  * 为代理 OkHttpClient 添加 `connectTimeout(15s)` 和 `readTimeout(30s)`，防止上游慢响应导致代理线程死锁。

## [V1.2.0] - 2026-05-26
### 🔄 终极流媒体代理架构突破
* **HLS 二级切片本地全量代理中转**：
  * 彻底消灭老电视由于系统 native 拉流引擎（如 Stagefright）不支持现代 TLSv1.2 加密、Let's Encrypt 证书过期或 IPv6 解析超时等缺陷导致的“解析直播源失败”。
  * 在本地 HTTP 代理网关 `LocalProxyServer` 中利用高精度的正则表达式，**动态将响应的 `.m3u8` 索引文件中所有的二级分片与子索引绝对路径 HTTPS 链接全部强行改写为本地回环代理地址**。
  * 保证老旧系统的 native 播放器只需与本地环回地址发起无任何加密与解密开销的纯 HTTP 请求，而所有复杂的解密、证书链信任与 IPv6 智能快乐眼球降级均在 Java 异步层由极其健壮的 OkHttp 客户端代劳，实现老旧盒子 100% 流畅秒开 HTTPS 直播流！

## [V1.1.0] - 2026-05-26
### 🛠️ 老旧系统 HTTPS 握手突破与交互优化
* **老盒子全局 TLS 1.2 连接补丁**：
  * 在 `MainActivity.java` 中嵌入并在 `onCreate()` 阶段向 `HttpsURLConnection` 注入 `TLSSocketFactory` 全局安全补丁类。在 Android 4.4 以下设备上强制开启并启用系统被默认禁用的 **TLS 1.1 与 TLS 1.2 握手协议支持**，以极其轻量的方式突破所有现代 HTTPS 直播源的握手失败问题。
* **侧边栏焦点动态防抢占机制**：
  * 在 `showSidebar()` 时，**动态彻底剥夺全屏播放器 `player_view` 的焦点竞争权**（`setFocusable(false)`），确保遥控器指针在菜单显示瞬间 100% 锁死在频道列表（`listChannels`）上；并在隐藏菜单时完整归还（`setFocusable(true)`），**完美解决首次打开目录选台无法操作的痛点**。
* **跨分类同名频道智能聚合**：
  * 重构 `M3uParser.java` 去重 Key 为单纯的 `currentName`。只要频道名一致，不论在哪个分类或哪一行，均 100% 强力聚合为同一频道下的多线路（线路 1/2/3...），净化列表并完美找回并倍增了所有的“多源链接”。
* **换源延时上调**：
  * 将 `setupPlayer()` 中自动换线的倒计时上调为 `3000ms`（3秒），为老电视的网速缓冲和连接转换留出更充裕的时间。

## [V1.0.0] - 2026-05-26
### 📦 初始自适应重构与层级净化
* **移除了布局文件中硬编码的 focusable 属性**：
  * 摒弃了老版脆弱的设备类型动态识别代码，转型为 Android 原生的 Touch Mode (触摸模式) 自适应机制，使 APP 既能完美高亮遥控操作电视盒子，又保障了手机端 100% 秒按秒响应触屏操作。
* **消除黑屏有声音故障**：
  * 将 `activity_main.xml` 中的 `player_view` 播放视图的 `app:surface_type` 修改为 `texture_view`。由于全局开启了硬件加速，`TextureView` 像常规视图一样正常参与布局层级树的重叠绘制，完美呈递在全屏 FrameLayout 的黑色背景之**上**，彻底斩断了 SurfaceView 被背景图层遮盖引起的黑屏宿疾。
* **抗灾级零依赖 GitHub Actions 自动编译**：
  * 重构 `.github/workflows/android.yml`，通过直接利用虚拟机本地预装的 JDK 11 以及官方 CDN 手动部署配置 Gradle 7.4，彻底摆脱了联网第三方 Action 的下载依赖，实现了在云端 1 分 30 秒内极速编译成功产出 debug APK。
