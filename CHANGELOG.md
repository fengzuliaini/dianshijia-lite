# 更新日志 (CHANGELOG)

本项目专注于为老电视及机顶盒（Android 4.1+, API 16+）与智能触屏手机提供完美的兼容性、流畅的遥控与触屏操作、以及极致的流媒体播放体验。

---

## [V1.2.0] - 2026-05-26 (当前版本)
### 🔄 终极流媒体代理架构突破
* **HLS 二级切片本地全量代理中转**：
  * 彻底消灭老电视由于系统 native 拉流引擎（如 Stagefright）不支持现代 TLSv1.2 加密、Let's Encrypt 证书过期或 IPv6 解析超时等缺陷导致的“解析直播源失败”。
  * 在本地 HTTP 代理网关 `LocalProxyServer` 中利用高精度的正则表达式，**动态将响应的 `.m3u8` 索引文件中所有的二级分片与子索引绝对路径 HTTPS 链接全部强行改写为本地回环代理地址**。
  * 保证老旧系统的 native 播放器只需与本地环回地址发起无任何加密与解密开销的纯 HTTP 请求，而所有复杂的解密、证书链信任与 IPv6 智能快乐眼球降级均在 Java 异步层由极其健壮的 OkHttp 客户端代劳，实现老旧盒子 100% 流畅秒开 HTTPS 直播流！

## [V1.1.0] - 2026-05-26
### 🛠️ 老旧系统 HTTPS 握手突破与交互优化
* **老盒子全局 TLS 1.2 连接补丁**：
  * 在 `MainActivity.java` 中嵌入并在 `onCreate()` 阶段向 `HttpsURLConnection` 注入 `TLSSocketFactory` 全局安全补丁类。在 Android 4.4 以下设备上强制开启并启用系统被默认禁用的 **TLS 1.1 与 TLS 1.2 握手协议支持**，以极其轻量的方式突破所有现代 HTTPS 直播源的握手失败问题。
* **侧边栏焦点动态防抢占机制**：
  * 在 `showSidebar()` 时，**动态彻底剥夺全屏播放器 `player_view` 的焦点竞争权**（`setFocusable(false)`），确保遥控器指针在菜单显示瞬间 100% 锁死在频道列表（`listChannels`）上；并在隐藏菜单时完整归还（`setFocusable(true)`），**完美解决首次打开目录选台无法操作的痛点**。
* **跨分类同名频道智能聚合**：
  * 重构 `M3uParser.java` 去重 Key 为单纯的 `currentName`。只要频道名一致，不论在哪个分类或哪一行，均 100% 强力聚合为同一频道下的多线路（线路 1/2/3...），净化列表并完美找回并倍增了所有的“多源链接”。
* **换源延时上调**：
  * 将 `setupPlayer()` 中自动换线的倒计时上调为 `3000ms`（3秒），为老电视的网速缓冲和连接转换留出更充裕的时间。

## [V1.0.0] - 2026-05-26
### 📦 初始自适应重构与层级净化
* **移除了布局文件中硬编码的 focusable 属性**：
  * 摒弃了老版脆弱的设备类型动态识别代码，转型为 Android 原生的 Touch Mode (触摸模式) 自适应机制，使 APP 既能完美高亮遥控操作电视盒子，又保障了手机端 100% 秒按秒响应触屏操作。
* **消除黑屏有声音故障**：
  * 将 `activity_main.xml` 中的 `player_view` 播放视图的 `app:surface_type` 修改为 `texture_view`。由于全局开启了硬件加速，`TextureView` 像常规视图一样正常参与布局层级树的重叠绘制，完美呈递在全屏 FrameLayout 的黑色背景之**上**，彻底斩断了 SurfaceView 被背景图层遮盖引起的黑屏宿疾。
* **抗灾级零依赖 GitHub Actions 自动编译**：
  * 重构 `.github/workflows/android.yml`，通过直接利用虚拟机本地预装的 JDK 11 以及官方 CDN 手动部署配置 Gradle 7.4，彻底摆脱了联网第三方 Action 的下载依赖，实现了在云端 1 分 30 秒内极速编译成功产出 debug APK。
